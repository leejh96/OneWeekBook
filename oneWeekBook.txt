일주일책

Onboard
로그인, 로그아웃, 회원가입, 비밀번호 찾기 기능
회원가입이나 비밀번호 찾기 시에 전화번호 인증
로그인: 이메일, 비밀번호
로그인 api { email: test@example.com, password: string }
로그인시 refreshToken, accessToken, user_data  발급

회원가입: 이메일, 비밀번호, 이름, 휴대폰 인증
이메일 확인 api { email: test@example.com }
휴대폰 인증 api (아래 설명 참조)
회원가입 api ( email: test@example.com, password: string, name: string, authToken: djawjpawfjpoawjfpaowjgpoawjfawjd }
회원가입시 refreshToken, accessToken, user_data  발급

비밀번호 찾기: 휴대폰 인증
휴대폰 인증 api (아래 설명 참조)
휴대폰 인증 시 임시 비밀번호 문자로 발급 or 새 비밀번호 입력

휴대폰 인증 시(조건 부여할 것[register, reset, change])
register: 회원 가입 휴대폰 인증 시 이용 (이미 가입된 휴대폰이면 인증 막기)
reset: (이미 가입된 휴대폰 중에서 비밀번호 찾기 시 이용)
change: (이미 가입된 휴대폰 중에서 비밀번호 변경 시 이용)

인증 절차
1. 휴대폰 번호 입력 (post) { phone: 01012341234 } 
2. 인증 코드와 토큰 발급 { code: 321321, codeToken: ajfdoawhfoiahwpwahprpawpwajpj }
3. 인증코드 입력(post) { code: 321321, codeToken: ajfdoawhfoiahwpwahprpawpwajpj }
4. 토큰 발급 { authToken: dwpojd[pawj[fjaw[fjaw[fjwa[fja }

기능
크게 도서 검색 기능과 도서 리뷰 기능, 리뷰 공유 기능으로 나뉘어 짐

도서 검색 기능 (Naver Api)
도서 검색하는 기능을 제공해준다.
해당 도서와 관련된 간단한 정보 제공 (검색 조건에 대해서는 토의 해볼 것)

도서 리뷰 기능
내가 읽은 책에 대한 리뷰를 작성한다. (CRUD)
작성하는 건 마크다운 편집기 이용 계획
어떤 걸 넣을지 세부적인 계획 필요

리뷰 공유 기능
내가 쓴 리뷰를 다른 사람들과 공유한다.
리뷰 공유를 누르면 다른 사람들이 해당 책의 자신이 쓴 리뷰를 볼 수 있게 해줌
어떻게 할지 세부적인 계획 필요
